# Otto Glossary

Authoritative terminology. UI label wins. Don't invent synonyms; use what's here.

- **Project** — Logical grouping of workspaces sharing a git remote (or main repo root). UI: "Project" / "Add project". Code: `ProjectSummary` (`packages/app/src/utils/projects.ts`), `projectKey` (`packages/server/src/server/workspace-registry-model.ts`). Forbidden: "Repo", "Repository" as UI label.
- **Workspace** — One concrete `cwd` on one daemon, with git state; belongs to exactly one project. Its `id` is opaque workspace identity; its `cwd` is the filesystem directory. UI: "Workspace". Code: `WorkspaceDescriptorPayload` (`packages/protocol/src/messages.ts`). Don't confuse with: Branch (one branch can back many workspaces via worktrees). Forbidden: "Folder", "Directory" as UI label.
- **Archive workspace** — Removes one workspace from active use and archives everything it owns. UI and app shortcuts always say "Archive workspace", regardless of backing. The daemon leaves ordinary directories intact and removes an Otto-owned worktree only when no active workspace still references it. CLI/MCP **archive worktree** is a separate lower-level operation that archives every workspace on that worktree.
- **Workspace kind** — `"directory" | "local_checkout" | "worktree"`. The git-derived, persisted property of a workspace, used across its lifetime (archive safety, sidebar, grouping). Derived from the cwd's git reality (`deriveWorkspaceKind`, `packages/server/src/server/workspace-registry-model.ts`), not stored from a user choice. Code: `PersistedWorkspaceKind` (`packages/server/src/server/workspace-registry-model.ts`). Don't confuse with **Isolation** (the create-time intent).
- **Isolation** — Create-time choice for a new workspace: reuse the existing checkout (**Local**) or cut a dedicated git worktree (**New worktree**). A transient setup input, also remembered as a create-form preference; it is not a workspace property. UI: "Isolation" control on the New Workspace screen. Code: `isolation` (`"local" | "worktree"`), `useWorkspaceIsolation` (`packages/app/src/screens/new-workspace-screen.tsx`); persisted as `FormPreferences.isolation` (`packages/app/src/create-agent-preferences/preferences.ts`). Distinct from **Workspace kind**, which is the git-derived property the intent produces (Local → `local_checkout` or `directory` by git-ness; New worktree → `worktree`). On the wire it is the create request's `source.kind` (`directory | worktree`, `packages/protocol/src/messages.ts`).
- **Agent** — See **Agent session**. UI still says "Agent" in technical/settings contexts (providers, personalities, permissions), but the conversation-creation action is **New chat** and the conversation surface reads "chat" (rename chat, chat not found). Code: `AgentSnapshotPayload` (`packages/protocol/src/messages.ts`). Forbidden: "Task", "Job", "Run".
- **New chat** — The action that creates a new agent session in a workspace. UI: "New chat" everywhere (command center, workspace header, tab actions/fallbacks, draft panel). Forbidden as UI labels: "New agent", "Create agent". Code identifiers stay historical: i18n keys `newAgent`, shortcut id `new-agent`, testIDs.
- **Add to chat** — The action that attaches content to the focused chat's composer as an attachment pill: files and folders (file explorer, Changes context menu), search hits and lines (project search), and PR comments/reviews/threads/check logs (PR panel). One label for all of them; the inverse is "Remove from chat". Forbidden as UI label: "Add to context". Code identifiers stay historical: i18n keys `addToContext`/`removeFromContext`, attachment kind `file_context`, store `useWorkspaceAttachmentsStore` (`packages/app/src/attachments/workspace-attachments-store.ts`).
- **Daemon** — Local Otto server process; identified by `serverId`. UI: "Daemon" (system contexts only). Code: `serverId` in `ServerInfoStatusPayloadSchema` (`packages/protocol/src/messages.ts`), `DaemonClient` (`packages/client/src/daemon-client.ts`).
- **Host** — Client-side connection profile pointing at a daemon; bundles one or more `HostConnection`s. UI: "Host" / "Add host" / "Switch host". Code: `HostProfile` (`packages/app/src/types/host-connection.ts`). Forbidden: "Connection" (means `HostConnection`, not host).
- **Project host entry** — One row in a project for a single (project, daemon) pair, aggregating that daemon's workspaces in the project. Internal. Code: `ProjectHostEntry` (`packages/app/src/utils/projects.ts`). Don't introduce "Checkout" as a synonym.
- **Placement** — One workspace's relationship to its project (projectKey, projectName, git checkout snapshot). Internal. Code: `ProjectPlacementPayload` (`packages/protocol/src/messages.ts`).
- **Branch** — Plain git branch. UI: "Switch branch". Code: `currentBranch` in `WorkspaceGitRuntimePayloadSchema` (`packages/protocol/src/messages.ts`); `BranchSwitcher` (`packages/app/src/components/branch-switcher.tsx`).
- **Worktree** — Otto-managed git worktree (`~/.otto/worktrees/{name}`); also a `workspaceKind` value. UI: CLI + `otto.json` keys (`worktree.setup`, `worktree.teardown`) only. Code: `ProjectCheckoutLiteGitOttoPayload` (`packages/protocol/src/messages.ts`); CLI `otto worktree` (`packages/cli/src/commands/worktree/index.ts`). Forbidden: "Checkout" as a synonym.
- **Repository / Remote** — Internal git inputs (`remoteUrl`, `mainRepoRoot`) used to derive `projectKey`. No UI label.
- **Git provider** (a.k.a. git hosting provider / forge) — The service backing a workspace's PR and issue features: **GitHub** or **Bitbucket Cloud**. Configured once per host under Host settings → Workspaces → "Git providers"; a workspace's provider is auto-derived from its remote (`github.com` → GitHub, `bitbucket.org` → Bitbucket Cloud). UI: "Git providers". Code: `GitHostingProviderId` (`packages/protocol/src/git-hosting.ts`), `GitHostingService` + `resolver.ts` (`packages/server/src/services/git-hosting/`). See [git-providers.md](git-providers.md). Forbidden: "GitHub" as the generic label now that Bitbucket exists.
- **Directory-backed surface** — A right-sidebar surface whose content is determined by the workspace's `cwd`, so two workspaces on the same directory see identical content: git diff/status, PR info (GitHub or Bitbucket Cloud, resolved from the cwd's remote), file preview/explorer contents. Keyed by `(serverId, cwd)`, never `workspaceId`. See [architecture.md](architecture.md#right-sidebar-boundary-directory-backed-vs-workspace-owned).
- **Workspace-owned state** — Per-workspace state that never leaks to a same-`cwd` sibling: tabs, agents, terminals, panes, title, plus review drafts, diff-mode overrides, composer attachments, and file-explorer open/expand state. Keyed by `workspaceId` (`cwd` only as a fallback for old payloads). See [architecture.md](architecture.md#right-sidebar-boundary-directory-backed-vs-workspace-owned).
- **Workspace status bucket** — Aggregate activity signal for a workspace row, computed strictly per `workspaceId`: an agent or terminal contributes only to the workspace it carries, and a terminal with no `workspaceId` contributes to nothing. Same-`cwd` siblings do not share status buckets (`applyTerminalContributions`, `packages/server/src/server/workspace-directory.ts`). Only git facts (branch, diff, dirty, PR) fan out across same-`cwd` workspaces — see **Directory-backed surface**.
- **Agent session** — One running instance of an agent inside a workspace (one provider, one model, one cwd, one timeline). The conceptual unit; in the UI this opens as a tab. Moving toward this as the canonical term over "Agent". Code: `AgentSnapshotPayload` (`packages/protocol/src/messages.ts`).
- **Session** — Two senses: (a) per-client connection to a daemon, internal; (b) user-facing agent session, see **Agent session**. Code: `Session` (`packages/server/src/server/session.ts`) for (a). Don't confuse with: provider-side agent session log.
- **Profile** — Internal name for the persisted shape of a host. Code: `HostProfile` (`packages/app/src/types/host-connection.ts`). Never user-facing.
- **Provider** — Agent backend (Claude Code, Codex, Copilot, OpenCode, Pi, OMP). UI: "Provider". Code: `ProviderSnapshotEntry` (`packages/protocol/src/messages.ts`).
- **Model** — A specific LLM offered by a provider. UI: "Model" / "Select model". Code: `AgentModelDefinition` (`packages/protocol/src/messages.ts`).
- **Personality** — A named, reusable per-host agent template binding a provider→model, effort, mode, prompt, one or more **roles**, and an identity (name, spinner colors, voice). The ergonomic "who does the work" pick, shown atop every model picker. Bound by stable `id`, never `name`. UI: "Agent personalities" (Host settings → Agents). Code: `AgentPersonality`, `PERSONALITY_ROLES` (`packages/protocol/src/messages.ts`); role/availability helpers in `packages/protocol/src/agent-personalities.ts`; resolution/snapshot in `packages/server/src/server/agent/agent-personalities.ts`. See [agent-personalities.md](agent-personalities.md). Don't confuse with **Provider** (a personality selects one) or **Mode** (a personality carries a default one).
- **Role** — One of 8 tags a personality carries (Chatter, Artificer, Scheduler, Writer, Coder, Judger, Advisor, Orchestrator) that gates where it appears and what it may do; only an **Orchestrator** may enumerate and spawn other personalities. Writer (fast small-text generation) and Coder (coding sub-agent) replaced the old single Worker role — a persisted `worker` tag resolves to `coder`. Code: `PersonalityRole` (`packages/protocol/src/messages.ts`). Don't confuse with **Mode** (permission level) or provider "categories".
- **Tab** — UI surface representing one session inside a workspace. Not a conceptual unit; use **Agent session** when talking about the model. Code: `WorkspaceTabDescriptor` (`packages/app/src/screens/workspace/workspace-tabs-types.ts`).
- **Terminal** — Workspace-scoped PTY shell streamed over the binary mux channel. UI: "Terminal". Code: `TerminalStreamFrame` (`packages/protocol/src/terminal-stream-protocol.ts`).
- **Schedule** — Cron-style trigger that creates new agents. UI: CLI/MCP (`otto schedule`, `create_schedule`). Don't confuse with: Heartbeat (cron prompt back into the same agent) or Loop (iterative re-execution of one agent).
- **Heartbeat** — Cron-style prompt sent back into the same agent/conversation. MCP: `create_heartbeat`. Use for reminders and babysitting where the status should return inline.
- **Orchestration** — UI label for a daemon-owned "Run": one execution of a multi-agent plan. Two flavors: **AI** (an orchestrator-role agent declares typed phases via `start_run` — or the user prompts one from the New Orchestration dialog) and **Graph** (a user-authored deterministic **Graph** executed exactly as drawn; `Run.kind: "graph"`). UI: "Orchestrations" (sidebar entry, page title), "New Orchestration" (dialog). Code stays `Run`/`RunPhase` (`packages/protocol/src/orchestration.ts`), `useRuns` (`packages/app/src/hooks/use-runs.ts`), `runs-screen.tsx`, `runs.*` RPCs — only the user-facing label changed. Don't confuse with **Agent session** (forbidden synonym "Run" there refers to that, not this).
- **Graph** — The reusable template a Graph-flavor **Orchestration** executes: agent nodes carrying roles + prompts, wired by edges, parameterized by declared inputs, with a single Orchestrator root that hosts the chat. Host-level and generic — an orchestration binds a graph to a workspace only at start time; executing a graph starts an orchestration. Authored in the **graph designer** (workspace tab, web/desktop only). UI: "Graph", "Graph designer". Code: `OrchestrationGraph` (`packages/protocol/src/orchestration.ts`), `GraphStore` (`$OTTO_HOME/orchestration-graphs/`), `graph-engine.ts`, tab kind `orchestrationGraph`. See `projects/orchestration-graphs/`. Forbidden: "workflow" (that's the Claude provider's Workflow tool), "blueprint", "template" as UI labels.
- **Mode** — Provider-specific operational mode (plan, default, full-access, …). UI: icon-only. Code: `modeId` in `AgentSessionConfig` (`packages/protocol/src/messages.ts`). Don't confuse with **Interface mode** (client display depth).
- **Interface mode** — The device-local display depth: **User** (a reduced view for chatting with agents and organizing projects — developer tools like files, diffs, terminals, git, and search are hidden) or **Developer** (the full environment, identical to today's app). Presentation only — never synced to the daemon, never changes what agents can do. UI: "Interface mode" (Settings → General; chosen first in the setup wizard) — always that two-word label, never bare "mode". Code: `interfaceMode: InterfaceMode | null` in `AppSettings` (`null` ⇒ Developer), gate `useIsDeveloperMode()` (`packages/app/src/hooks/use-interface-mode.ts`). See [../projects/first-time-wizard/interface-modes.md](../projects/first-time-wizard/interface-modes.md). Forbidden labels: "Simple mode", "Basic mode", "Pro mode". Don't confuse with **Mode** (provider operational mode).
- **Effort** — How hard the model reasons before answering; one concept across all providers even though vendors brand it differently (Claude "thinking", OpenAI "reasoning effort") and option sets vary per model/provider (on/off, low/medium/high, …). Options are advertised per model via `thinkingOptions` on `AgentModelDefinition`; never force a fixed option set. UI: "Effort" / "Select effort" everywhere (composer control, schedule form, artifact form). Tooling/prompts request effort via the canonical level scale `off < minimal < low < medium < high < xhigh < max`, resolved to the nearest option the target model supports (`resolveEffortOption`, `packages/server/src/server/agent/effort-levels.ts`); exact option ids always win, keeping provider-special options (Claude `ultracode`) reachable. Code identifiers are historical and stay: `thinkingOptionId`, `thinkingOptions`, `thinkingByModel`, i18n keys `agentControls.thinking.*`, testIDs `*-thinking-*`. Forbidden as UI labels: "Thinking", "Reasoning". Don't confuse with **reasoning blocks** (the chain-of-thought text an agent streams, e.g. the "Always expand reasoning" setting — that stays "reasoning").
- **Attachment** — A PR or issue bound to an agent prompt, from any configured **git provider** (GitHub or Bitbucket Cloud). UI: "Attach issue or PR". Code: `AgentAttachment` (`packages/protocol/src/messages.ts`); provider-neutral kinds `hosting_pr` / `hosting_issue` (each carries a `provider`), with legacy `github_pr` / `github_issue` still accepted. See [git-providers.md](git-providers.md).
- **Composer** — The whole prompt surface for sending work to an agent. Code: `Composer` (`packages/app/src/composer/index.tsx`). Don't call this "message input" except for the text-entry subcomponent.
- **Composer input** — The text-entry surface inside the composer. Code: `MessageInput` (`packages/app/src/composer/input/input.tsx`).
- **Composer toolbar** — The bottom control row inside the composer input. Contains agent controls, attachment button, voice controls, and stop/send controls. Code: `leftContent`, `beforeVoiceContent`, and `rightContent` slots in `MessageInput` (`packages/app/src/composer/input/input.tsx`). Forbidden: "Status bar".
- **Agent controls** — Provider, model, mode, thinking, and provider-feature controls for an agent or draft agent. Code: `AgentControls` / `DraftAgentControls` (`packages/app/src/composer/agent-controls/index.tsx`). Forbidden: "Agent status bar".
- **Composer footer** — Optional area rendered below the composer input but still inside the keyboard-shifted composer layout. Code: `Composer.footer` (`packages/app/src/composer/index.tsx`).
- **Composer track** — A contextual lane above the composer input. Specific tracks use the `<thing> track` form: **Queue track**, **Subagents track**, **Background Tasks track**. Code: queue track inside `Composer` (`packages/app/src/composer/index.tsx`), `SubagentsTrack` (`packages/app/src/subagents/track.tsx`), `BackgroundTasksTrack` (`packages/app/src/background-tasks/track.tsx`).
- **Background Tasks track** — Lists background shell processes a provider launched itself (Claude Bash tool `run_in_background`) — plain shell commands, never AI subagents. Renders independently of and alongside the Subagents track; a row's actions are Stop (while running) and Clear (once terminal). Code: `BackgroundTasksTrack` (`packages/app/src/background-tasks/track.tsx`). Forbidden: "background jobs", "shell tasks", "background agents" (implies AI).
- **Attachment tray** — The selected-attachments row inside the composer input, above the text input. Code: `renderAttachmentTray` (`packages/app/src/composer/index.tsx`). Forbidden: "Attachment bar".
- **Conflict** — Two distinct senses; do NOT use the bare word in UI copy without qualifying which: (a) **stale-write conflict** on `otto.json` ("Config changed on disk", code `stale_project_config`, `packages/app/src/screens/project-settings-screen.tsx`); (b) **git merge conflict** (no current UI string).

## Inconsistencies (documented, not papered over)

- CLI `--host <host>` description `"Daemon host target"` (`packages/cli/src/utils/command-options.ts`) blurs daemon/host; the app keeps them distinct.
- `WorkspaceDescriptorPayloadSchema.workspaceKind` accepts legacy `"checkout"` on the wire (`packages/protocol/src/messages.ts`) while `PersistedWorkspaceKind` does not (`packages/server/src/server/workspace-registry-model.ts`).
