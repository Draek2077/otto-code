# Otto Glossary

Authoritative terminology. UI label wins. Don't invent synonyms; use what's here.

- **Project** ‚Äî Logical grouping of workspaces sharing a git remote (or main repo root). UI: "Project" / "Add project". Code: `ProjectSummary` (`packages/app/src/utils/projects.ts`), `projectKey` (`packages/server/src/server/workspace-registry-model.ts`). Forbidden: "Repo", "Repository" as UI label.
- **Workspace** ‚Äî One concrete `cwd` on one daemon, with git state; belongs to exactly one project. Its `id` is opaque workspace identity; its `cwd` is the filesystem directory. UI: "Workspace". Code: `WorkspaceDescriptorPayload` (`packages/protocol/src/messages.ts`). Don't confuse with: Branch (one branch can back many workspaces via worktrees). Forbidden: "Folder", "Directory" as UI label.
- **Workspace kind** ‚Äî `"directory" | "local_checkout" | "worktree"`. The git-derived, persisted property of a workspace, used across its lifetime (archive safety, sidebar, grouping). Derived from the cwd's git reality (`deriveWorkspaceKind`, `packages/server/src/server/workspace-registry-model.ts`), not stored from a user choice. Code: `PersistedWorkspaceKind` (`packages/server/src/server/workspace-registry-model.ts`). Don't confuse with **Isolation** (the create-time intent).
- **Isolation** ‚Äî Create-time choice for a new workspace: reuse the existing checkout (**Local**) or cut a dedicated git worktree (**New worktree**). A transient setup input, also remembered as a create-form preference; it is not a workspace property. UI: "Isolation" control on the New Workspace screen. Code: `isolation` (`"local" | "worktree"`), `useWorkspaceIsolation` (`packages/app/src/screens/new-workspace-screen.tsx`); persisted as `FormPreferences.isolation` (`packages/app/src/create-agent-preferences/preferences.ts`). Distinct from **Workspace kind**, which is the git-derived property the intent produces (Local ‚Üí `local_checkout` or `directory` by git-ness; New worktree ‚Üí `worktree`). On the wire it is the create request's `source.kind` (`directory | worktree`, `packages/protocol/src/messages.ts`).
- **Agent** ‚Äî See **Agent session**. UI still says "Agent" / "New Agent" in places, but moving toward **Agent session** as the canonical term. Code: `AgentSnapshotPayload` (`packages/protocol/src/messages.ts`). Forbidden: "Task", "Job", "Run".
- **Daemon** ‚Äî Local Otto server process; identified by `serverId`. UI: "Daemon" (system contexts only). Code: `serverId` in `ServerInfoStatusPayloadSchema` (`packages/protocol/src/messages.ts`), `DaemonClient` (`packages/client/src/daemon-client.ts`).
- **Host** ‚Äî Client-side connection profile pointing at a daemon; bundles one or more `HostConnection`s. UI: "Host" / "Add host" / "Switch host". Code: `HostProfile` (`packages/app/src/types/host-connection.ts`). Forbidden: "Connection" (means `HostConnection`, not host).
- **Project host entry** ‚Äî One row in a project for a single (project, daemon) pair, aggregating that daemon's workspaces in the project. Internal. Code: `ProjectHostEntry` (`packages/app/src/utils/projects.ts`). Don't introduce "Checkout" as a synonym.
- **Placement** ‚Äî One workspace's relationship to its project (projectKey, projectName, git checkout snapshot). Internal. Code: `ProjectPlacementPayload` (`packages/protocol/src/messages.ts`).
- **Branch** ‚Äî Plain git branch. UI: "Switch branch". Code: `currentBranch` in `WorkspaceGitRuntimePayloadSchema` (`packages/protocol/src/messages.ts`); `BranchSwitcher` (`packages/app/src/components/branch-switcher.tsx`).
- **Worktree** ‚Äî Otto-managed git worktree (`~/.otto/worktrees/{name}`); also a `workspaceKind` value. UI: CLI + `otto.json` keys (`worktree.setup`, `worktree.teardown`) only. Code: `ProjectCheckoutLiteGitOttoPayload` (`packages/protocol/src/messages.ts`); CLI `otto worktree` (`packages/cli/src/commands/worktree/index.ts`). Forbidden: "Checkout" as a synonym.
- **Repository / Remote** ‚Äî Internal git inputs (`remoteUrl`, `mainRepoRoot`) used to derive `projectKey`. No UI label.
- **Directory-backed surface** ‚Äî A right-sidebar surface whose content is determined by the workspace's `cwd`, so two workspaces on the same directory see identical content: git diff/status, GitHub PR info, file preview/explorer contents. Keyed by `(serverId, cwd)`, never `workspaceId`. See [architecture.md](architecture.md#right-sidebar-boundary-directory-backed-vs-workspace-owned).
- **Workspace-owned state** ‚Äî Per-workspace state that never leaks to a same-`cwd` sibling: tabs, agents, terminals, panes, title, plus review drafts, diff-mode overrides, composer attachments, and file-explorer open/expand state. Keyed by `workspaceId` (`cwd` only as a fallback for old payloads). See [architecture.md](architecture.md#right-sidebar-boundary-directory-backed-vs-workspace-owned).
- **Workspace status bucket** ‚Äî Aggregate activity signal for a workspace row, computed strictly per `workspaceId`: an agent or terminal contributes only to the workspace it carries, and a terminal with no `workspaceId` contributes to nothing. Same-`cwd` siblings do not share status buckets (`applyTerminalContributions`, `packages/server/src/server/workspace-directory.ts`). Only git facts (branch, diff, dirty, PR) fan out across same-`cwd` workspaces ‚Äî see **Directory-backed surface**.
- **Agent session** ‚Äî One running instance of an agent inside a workspace (one provider, one model, one cwd, one timeline). The conceptual unit; in the UI this opens as a tab. Moving toward this as the canonical term over "Agent". Code: `AgentSnapshotPayload` (`packages/protocol/src/messages.ts`).
- **Session** ‚Äî Two senses: (a) per-client connection to a daemon, internal; (b) user-facing agent session, see **Agent session**. Code: `Session` (`packages/server/src/server/session.ts`) for (a). Don't confuse with: provider-side agent session log.
- **Profile** ‚Äî Internal name for the persisted shape of a host. Code: `HostProfile` (`packages/app/src/types/host-connection.ts`). Never user-facing.
- **Provider** ‚Äî Agent backend (Claude Code, Codex, Copilot, OpenCode, Pi, OMP). UI: "Provider". Code: `ProviderSnapshotEntry` (`packages/protocol/src/messages.ts`).
- **Model** ‚Äî A specific LLM offered by a provider. UI: "Model" / "Select model". Code: `AgentModelDefinition` (`packages/protocol/src/messages.ts`).
- **Tab** ‚Äî UI surface representing one session inside a workspace. Not a conceptual unit; use **Agent session** when talking about the model. Code: `WorkspaceTabDescriptor` (`packages/app/src/screens/workspace/workspace-tabs-types.ts`).
- **Terminal** ‚Äî Workspace-scoped PTY shell streamed over the binary mux channel. UI: "Terminal". Code: `TerminalStreamFrame` (`packages/protocol/src/terminal-stream-protocol.ts`).
- **Schedule** ‚Äî Cron-style trigger that creates new agents. UI: CLI/MCP (`otto schedule`, `create_schedule`). Don't confuse with: Heartbeat (cron prompt back into the same agent) or Loop (iterative re-execution of one agent).
- **Heartbeat** ‚Äî Cron-style prompt sent back into the same agent/conversation. MCP: `create_heartbeat`. Use for reminders and babysitting where the status should return inline.
- **Mode** ‚Äî Provider-specific operational mode (plan, default, full-access, ‚Ä¶). UI: icon-only. Code: `modeId` in `AgentSessionConfig` (`packages/protocol/src/messages.ts`).
- **Effort** ‚Äî How hard the model reasons before answering; one concept across all providers even though vendors brand it differently (Claude "thinking", OpenAI "reasoning effort") and option sets vary per model/provider (on/off, low/medium/high, ‚Ä¶). Options are advertised per model via `thinkingOptions` on `AgentModelDefinition`; never force a fixed option set. UI: "Effort" / "Select effort" everywhere (composer control, schedule form, artifact form). Tooling/prompts request effort via the canonical level scale `off < minimal < low < medium < high < xhigh < max`, resolved to the nearest option the target model supports (`resolveEffortOption`, `packages/server/src/server/agent/effort-levels.ts`); exact option ids always win, keeping provider-special options (Claude `ultracode`) reachable. Code identifiers are historical and stay: `thinkingOptionId`, `thinkingOptions`, `thinkingByModel`, i18n keys `agentControls.thinking.*`, testIDs `*-thinking-*`. Forbidden as UI labels: "Thinking", "Reasoning". Don't confuse with **reasoning blocks** (the chain-of-thought text an agent streams, e.g. the "Always expand reasoning" setting ‚Äî that stays "reasoning").
- **Attachment** ‚Äî GitHub PR or Issue bound to an agent prompt. UI: "Attach issue or PR". Code: `AgentAttachment` (`packages/protocol/src/messages.ts`).
- **Composer** ‚Äî The whole prompt surface for sending work to an agent. Code: `Composer` (`packages/app/src/composer/index.tsx`). Don't call this "message input" except for the text-entry subcomponent.
- **Composer input** ‚Äî The text-entry surface inside the composer. Code: `MessageInput` (`packages/app/src/composer/input/input.tsx`).
- **Composer toolbar** ‚Äî The bottom control row inside the composer input. Contains agent controls, attachment button, voice controls, and stop/send controls. Code: `leftContent`, `beforeVoiceContent`, and `rightContent` slots in `MessageInput` (`packages/app/src/composer/input/input.tsx`). Forbidden: "Status bar".
- **Agent controls** ‚Äî Provider, model, mode, thinking, and provider-feature controls for an agent or draft agent. Code: `AgentControls` / `DraftAgentControls` (`packages/app/src/composer/agent-controls/index.tsx`). Forbidden: "Agent status bar".
- **Composer footer** ‚Äî Optional area rendered below the composer input but still inside the keyboard-shifted composer layout. Code: `Composer.footer` (`packages/app/src/composer/index.tsx`).
- **Composer track** ‚Äî A contextual lane above the composer input. Specific tracks use the `<thing> track` form: **Queue track**, **Subagents track**. Code: queue track inside `Composer` (`packages/app/src/composer/index.tsx`), `SubagentsTrack` (`packages/app/src/subagents/track.tsx`).
- **Attachment tray** ‚Äî The selected-attachments row inside the composer input, above the text input. Code: `renderAttachmentTray` (`packages/app/src/composer/index.tsx`). Forbidden: "Attachment bar".
- **Conflict** ‚Äî Two distinct senses; do NOT use the bare word in UI copy without qualifying which: (a) **stale-write conflict** on `otto.json` ("Config changed on disk", code `stale_project_config`, `packages/app/src/screens/project-settings-screen.tsx`); (b) **git merge conflict** (no current UI string).

## Inconsistencies (documented, not papered over)

- CLI `--host <host>` description `"Daemon host target"` (`packages/cli/src/utils/command-options.ts`) blurs daemon/host; the app keeps them distinct.
- `WorkspaceDescriptorPayloadSchema.workspaceKind` accepts legacy `"checkout"` on the wire (`packages/protocol/src/messages.ts`) while `PersistedWorkspaceKind` does not (`packages/server/src/server/workspace-registry-model.ts`).
