# Task: Add i18n Strings for Artifacts UI

## Goal

Add translation keys for all user-visible strings in the artifacts feature.

## Context

Otto uses `react-i18next` for internationalization. All UI strings use `useTranslation()` and `t("key.path")`. The left sidebar already has `sidebar.sections.artifacts` as a placeholder (task 08).

## References

- `packages/app/src/i18n/` — existing translation files
- `packages/app/src/components/left-sidebar.tsx` — uses `t("sidebar.sections.schedules")` etc.
- All artifact components from tasks 08–18

## What to Do

### 1. Add Translation Keys

Add keys under a new `artifacts` namespace in the i18n resource files (follow the existing pattern, e.g., `en.json`, `zh.json`, etc.):

```json
{
  "artifacts": {
    "title": "Artifacts",
    "emptyTitle": "No artifacts yet",
    "emptyDescription": "Create your first artifact to get started",
    "create": "New Artifact",
    "createSuccess": "Artifact created successfully",
    "deleteConfirm": "Delete this artifact? This cannot be undone.",
    "deleteSuccess": "Artifact deleted",
    "open": "Open",
    "modify": "Modify",
    "modifyPrompt": "Describe how you want to modify this artifact",
    "retry": "Retry",
    "star": "Star",

      "generating": "Generating...",

    "errors": {
      "fileNotFound": "Artifact file not found",
      "generationTimeout": "Generation timed out",
      "malformedHtml": "This artifact contains invalid HTML. Showing raw content.",
      "projectNotFound": "Project not found"
    }
  }
}
```

### 2. Update Sidebar Key

Ensure `sidebar.sections.artifacts` exists in the sidebar section keys:

```json
{
  "sidebar": {
    "sections": {
      "sessions": "Sessions",
      "schedules": "Schedules",
      "artifacts": "Artifacts"
    }
  }
}
```

### 3. Replace Hardcoded Strings

In all artifact components (tasks 08–18), replace hardcoded strings with `t()` calls:

- `artifact-card.tsx` — status labels, button text, menu items, confirmation dialog
- `artifact-grid.tsx` — empty state text
- `artifact-create-sheet.tsx` — form labels, button text, error messages
- `artifact-panel.tsx` — error messages, loading text
- `left-sidebar.tsx` — "Artifacts" label
- `artifacts-screen.tsx` — screen title

## Acceptance Criteria

- All user-visible strings use `t()` translation keys
- Keys are added to the default language file (en)
- No hardcoded English strings remain in artifact components
- Follows the existing i18n file structure and naming conventions
