# Task 04 — polish + product integration

Prereqs: tasks 01–03 landed.

1. **open-file routing** — page sends `{type:'open-file', filePath, line?}`; route to `usePaneContext().openFileInWorkspace`. Paths from the page are workspace-relative or absolute as the adapter supplied them in tool `inputData.file_path` — feed paths in a form the file opener accepts (verify against the unified file tab's expectations, docs/text-editor.md).
2. **Panel-toggle settings** — device-local Visualizer settings (which page panels are on: timeline, file attention, transcript, message feed, cost overlay, hex grid) seeded via the `config` bridge message on attach. The page's `VisualizerConfig` only supports `{mode, autoPlay, showMockData, disable1MContext}` today — extending it means a vendor patch (log in `vendor/agent-flow/OTTO-PATCHES.md`) or, better, an upstream PR adding a `panels`/`theme` config object. Prefer upstreaming.
3. **Audio default-mute** — the page has sound effects (`use-audio-effects`); ensure the embed defaults muted (config or vendor patch; check what the control bar persists).
4. **Provider logos** — `web/components/agent-visualizer/canvas/draw-agents.ts` `drawAgentBrand` supports claude/codex only. Extend for Copilot/OpenCode/Pi/openai-compat via a small vendor patch or (preferred) an upstream PR making the runtime→logo mapping data-driven. Until then the default (claude glyph) is acceptable for other providers — or omit `runtime` and accept the default.
5. **Attribution** — Settings→About (or wherever third-party credits live): "Visualizer is derived from Agent Flow (Apache 2.0) by Simon Patole". Never ship upstream logos; never use the name "Agent Flow" as a feature label.
6. **Orchestration module (Runs) hookup** — when a Run (`features.agentOrchestration`, `packages/app/src/hooks/use-runs.ts`) is active, offer "Visualize" on the run: same adapter, session list = the run's agent set, so a team run renders as one multi-session view. Design with the agent-teams/orchestration owner; this is the product owner's stated end-goal.
7. **Docs fold-in** — when this ships, fold durable facts (vendor sync playbook, embed pattern, adapter architecture) into a new `docs/visualizer.md`, add the CLAUDE.md docs-table row, and delete `projects/visualizer/` per repo convention.
